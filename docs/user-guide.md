# 使用文档：商家 & 仓库 操作指南

本文档面向**商家（MERCHANT）**和**仓库员（WAREHOUSE_STAFF）**两类账户，按业务流程从头到尾走一遍**库存管理**和**订单管理**两条主线。管理员（ADMIN）的权限是商家 + 仓库员的并集，可以参照对应章节。

---

## 一、角色与责任

| 角色 | 主要责任 |
| --- | --- |
| **商家（MERCHANT）** | 提交入库单（让仓库知道有货要到）；同步、确认、取消平台订单 |
| **仓库员（WAREHOUSE_STAFF）** | 收货并核对入库单数量；为已确认的订单打印面单、发货；管理库存调整 |
| **管理员（ADMIN）** | 上述全部能力 + 替任意商家做导入、设置自动确认时长等 |

---

## 二、库存管理

库存的两个动作：**入库**（货物到仓）和**调整**（盘点、损耗、退货等）。

商家不直接改库存，必须通过**入库单**让仓库确认；仓库员可以直接调整或导入库存。

### 2.1 商家：提交入库单

> 适用场景：你给仓库发了一批货，需要在系统里登记，等仓库收到后核对入库。

**步骤：**

1. 登录后进入 **库存 → 立即导入**（页面右上角按钮）。
2. **下载模板**：点击紫色的「下载模板」按钮，得到一个 `.xlsx` 文件。模板里已经预置了你账户下的仓库和分类下拉。
3. **填写模板**：每行填 `仓库名 / SKU / 商品名 / 数量变化（+N）/ 事件类型 / 备注`。
   - **同一份文件里所有行必须发往同一个仓库**，否则提交时会被拒绝（请按仓库拆分文件分次提交）。
   - 数量填 **正数**：表示要入多少；不要填 0 或负数。
4. **上传文件**：把 `.xlsx` 拖到上传区，点击「预览变动」。
5. **核对预览**：系统会列出每一行的解析结果。红色 Tag 是错误行（如 SKU 不存在），绿色「将应用」是有效行。如有错误请修改模板再重新上传。
6. **填写发货信息**（下一步）：
   - 货物发出日期（必填，不能晚于今天）
   - 收货仓库（自动从预览推断，不可改）
   - 快递公司（如 J&T Express）
   - 快递单号
7. **提交入库单**。系统会把这单生成一条 `PENDING_REVIEW` 状态的入库单，**此时库存还没生效**。

**之后会发生什么：**

- 仓库员会在「库存 → 入库单」看到你这单。
- 仓库收货后核对实际数量，确认入库 → 库存数才会真正增加。
- 如果仓库填写的实收数量和你的预报不一致（少件、破损），系统会按**实收数量**入库。
- 如果仓库拒收，你会在「入库单」页看到 `REJECTED` 状态和拒收原因。

### 2.2 仓库员：确认/拒收入库单

> 适用场景：仓库收到了商家发来的货。

**步骤：**

1. 进入 **库存 → 入库单**。默认筛选 `待审核（PENDING_REVIEW）`，列表里就是所有等你处理的入库单。
2. 点击某一行的「审核」按钮，打开详情。
3. **核对实物**：对照页面上的 SKU 列表点实际数量。
   - 「预期数量」是商家报的；
   - 「确认数量」可编辑：默认等于预期，少件请改小，多件可以改大。
4. **二选一**：
   - **确认入库**：点右下角紫色「确认入库」→ 二次确认 → 库存按你填的确认数量入账，状态变 `CONFIRMED`。
   - **拒收**：点「拒收」按钮 → 弹窗里填**拒收原因**（必填）→ 提交。这一单作废，库存不变，状态变 `REJECTED`。
5. 已确认/拒收的单子可以在状态 Tab 里回查，但不能修改。

### 2.3 仓库员/管理员：直接导入库存

> 适用场景：盘点、初始化、修正历史误差等不需要走入库单的情况。

**步骤：**

1. 进入 **库存 → 立即导入**。
2. **选择归属商家**：因为库存按商家隔离，导入前必须先指定这批库存归谁所有。下拉里搜索选择即可。
3. 下载模板、填写、上传、预览（与商家流程相同）。
4. 点击「应用 N 行」→ 库存**立即生效**，不需要二次确认。
5. 完成后可在「库存 → 历史」查到本次变动记录。

### 2.4 库存查看与单条调整

- **库存列表**（`/inventory`）：按仓库、分类、商家、SKU、低库存筛选。
- **单条调整**：列表里每行有「调整」按钮，打开后可输入 +/- 数量和**原因**（必填，从下拉选）。适合处理零散的损耗、退货上架等。
- **历史**（`/inventory/history`）：所有变动的审计流水。

## 三、订单管理

订单从平台同步进来后，要先走「商家确认」环节，确认后仓库才看得到并能发货。

### 3.1 同步订单

订单**会自动同步**，不需要手动操作。后台机制：

- **定时任务**：每分钟自动同步一次所有已连接的店铺。
- **页面拉取**：每次进入「订单管理」页时，会触发一次后台同步；列表本身每 30 秒自动刷新一次。
- **手动同步**：右上角「立即同步」按钮，约 8 秒后列表刷新。如果你只想同步某一个店铺，可以先在筛选器里选中那家店铺再点同步。

### 3.2 商家：确认 / 取消订单

> 适用场景：买家下单后，订单进入「待商家确认」，你要决定接单还是拒单。

#### 单笔操作

1. 进入 **订单管理**。商家默认筛选条件是「待商家确认」。
2. 在订单行最右侧：
   - **绿色 ✓**：确认订单 → 二次确认 → 该订单的库存被预占，仓库能看到并开始备货。
   - **红色 ✗**：取消订单 → 二次确认 → 订单状态变 `CANCELLED`，已预占的库存释放回去。

#### 批量操作（多个订单一次处理）

1. 在表格最左侧勾选要操作的订单（**只有「待商家确认」的订单**才能勾选，其他订单的复选框是灰色的）。
2. 表格上方会出现操作栏：**已选 N 个订单**，旁边有「批量确认」和「批量取消」。
3. 点击需要的操作 → 弹窗里再次确认 → 系统逐个处理并显示进度条。
4. 完成后会提示「成功 X 个，失败 Y 个」（如果有失败的，原始单条记录可在订单详情里看到原因）。

#### 自动确认

如果商家长时间不处理，系统会**自动确认**：

- 默认窗口：**24 小时**（管理员可在用户设置里把 `autoConfirmHours` 改为 1～168 小时）。
- 后台每 **5 分钟** 扫一次过期未确认的订单，把它们置为 `AUTO_CONFIRMED`，等同于商家手动确认。
- 自动确认的订单在「确认状态」筛选器里能用 `已自动确认` 单独筛出。

> 想留出手动决策时间的商家，**请在窗口期内处理**，否则系统帮你确认掉就没法再回到「待商家确认」了。

### 3.3 仓库员：发货 / 打印面单

商家确认后，订单状态会进入「待发货」（`TO_SHIP / AWAITING_SHIPMENT / AWAITING_COLLECTION` 等），此时仓库可以打印面单。

#### 单张面单

订单行最右侧的 🖨 按钮：点一下 → 系统从平台拉运单 → 自动打开新窗口预览/打印。
如果订单被拆成了多个包裹，会自动拉回所有包裹的面单合并成一份 PDF。

#### 批量打印（推荐）

> 适用场景：一次性把今天所有要发的单都打出来。

1. 用顶部筛选器筛出你想打印的订单（如：店铺 + 状态「待发货」）。
2. 点页面右上角紫色「**批量打印**」按钮。
3. 弹窗会告诉你：
   - 即将打印 N 张面单；
   - 跳过 M 张不可打印的订单（已发货 / 已取消等）。
4. 确认后，系统逐个拉取面单（进度条显示 `X / N`），最后**合并成一份 PDF** 在新窗口打开，直接打印即可。
5. 失败的张数会单独提示，可以稍后重试或单张打印。

> 注意：批量打印涉及 TikTok 真实的快递安排（J&T 上门取件），**不可撤销**，确认前请核对筛选条件。

### 3.4 状态流转速查

```
平台下单
   │
   ▼
[UNPAID 待付款] ──支付成功──▶ [PENDING_CONFIRM 待商家确认]
                                       │
                          商家确认 / 自动确认
                                       │
                                       ▼
                          [TO_SHIP 待发货] ──打印面单/发货──▶ [IN_TRANSIT 运输中]
                                       │                                    │
                                       │                                    ▼
                                       │                            [DELIVERED 已送达]
                                       │                                    │
                                       ▼                                    ▼
                          [CANCELLED 已取消]                        [COMPLETED 已完成]
```

商家确认状态（独立于订单状态）：

| 值 | 含义 |
| --- | --- |
| `PENDING_CONFIRM` | 等商家点确认/取消 |
| `CONFIRMED` | 商家手动确认 |
| `AUTO_CONFIRMED` | 系统自动确认（超过 autoConfirmHours） |
| `CANCELLED_BY_MERCHANT` | 商家点了取消 |

## 四、常见问题

**Q：上传库存模板时报错「ownerUserId (merchant) is required」？**
你是仓库员/管理员账户，但没在导入对话框里选择归属商家。回到上传表单顶部，先选好商家再上传。

**Q：入库单提交时报错「所有行必须发往同一仓库」？**
你的 `.xlsx` 里出现了多个不同的仓库名。请按仓库拆成多份文件分别提交。

**Q：批量确认订单时，部分订单失败怎么办？**
失败一般是因为该订单已经被其他人（或自动确认）处理过了。打开订单详情查看当前状态；状态已经不是 `PENDING_CONFIRM` 的话就不用再处理了。

**Q：订单列表里看不到新订单？**
列表每 30 秒自动刷新一次，平台同步每分钟一次。如果着急可以点右上角「立即同步」，约 8 秒后刷新；同步只对当前筛选的店铺生效（不筛选时同步所有店铺）。

**Q：库存导入应用后想撤回？**
不能直接撤回，但可以通过「单条调整」用相反方向的数量做反向调整，并在原因里写明「修正 N 月 N 日导入误差」。所有动作都会留下审计记录。
